# Adaptive Structure

## Premise

The organization's structure is not a static diagram. It is a **continuously
re-fit topology** that adapts to environment variety, internal load, and
purpose drift.

## What Adapts

| Element | Adaptation cadence | Authority |
|---|---|---|
| Skills | Continuous (autonomous) | Self-promotion if criteria met |
| Agents | Weekly (proposed) | Coordination layer ratifies |
| Clusters | Monthly (proposed) | Governance system ratifies |
| Nervous systems | Quarterly (reviewed) | Human + governance ratification |
| Org core | Yearly (or on cause) | Human only |

This nested cadence ensures fast adaptation at the edges and slow stability
at the core.

## Adaptation Mechanisms

### 1. Pattern Promotion

When the same cognitive transaction recurs, it is promoted from ad-hoc to
skill, then potentially to agent, then potentially to cluster (see
`organizational_theory.md` lifecycle).

### 2. Pattern Demotion

Skills, agents, and clusters that fall below activity or quality thresholds
are demoted: archived to memory, replaced, or merged.

### 3. Topology Re-fit

The coordination layer periodically reanalyzes the event graph and proposes
re-bindings — which agents subscribe to which streams, which clusters share
context buses.

### 4. Capacity Reallocation

Compute, memory, and simulation budgets are continuously rebalanced across
subsystems based on observed load and outcome quality.

### 5. Architectural Mutations

Larger structural changes are proposed by the learning system as
**architectural diffs** — explicit before/after specs, simulation evidence,
expected outcomes, rollback plan.

## What Does Not Adapt Autonomously

- Purpose
- Ethical principles
- Autonomy ceiling
- Authority routing for irreversible action
- Identity invariants

These can change, but only via the human strategic layer.

## Stability Anchors

Adaptation can run away. We anchor it with:

- **Constitutional invariants** — purpose, ethics, values
- **Memory continuity** — adapted structures inherit memory; identity persists
- **Reversibility windows** — every adaptation is rollback-able for a window
- **Coherence audits** — adaptations that break coherence are retracted

## Failure Modes

- **Adaptation thrash** — too-frequent change destroys learning
- **Adaptation freeze** — gates too tight, environment outpaces structure
- **Local optima** — clusters adapt for cluster, not for org
- **Orphaned structures** — dead skills/agents not pruned

The learning system tracks each.

## Trade-offs

- Higher adaptation rate → higher responsiveness, lower stability.
- Slower adaptation rate → higher stability, drift risk.
- Greater architectural plasticity → more learning, harder governance.

We default to **slow at the core, fast at the edge**.
