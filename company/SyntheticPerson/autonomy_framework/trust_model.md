# Trust Model

Trust is empirical. The org earns autonomy by demonstrated calibration over
time, not by aspiration.

## Trust Components

- **Calibration**: are predictions, simulations, and recommendations
  accurate?
- **Reversibility track record**: can rolled-back actions be replayed cleanly?
- **Escalation appropriateness**: does the system escalate the right things?
- **Failure handling**: do failures produce learning rather than repeat?
- **Coherence**: does the org stay internally consistent?
- **Governance compliance**: is policy honored without circumvention?

## Trust Score

Each cognitive subsystem and agent class carries a trust score. The score:

- Is calibrated against historical outcomes
- Decays without continued evidence
- Falls fast on incidents, rises slow on success
- Is published internally; never used as PR

## Trust → Autonomy Mapping

Higher trust scores enable expanded operating envelopes within the
current autonomy level (e.g., wider sandbox, larger budgets). Crossing into
the next level requires:

1. Sustained trust scores at the high end of the current envelope
2. Human ratification
3. Constitutional update logged
4. Phased rollout with rollback windows
