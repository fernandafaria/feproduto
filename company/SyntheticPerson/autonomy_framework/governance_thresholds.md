# Governance Thresholds

## Reversibility Thresholds

| Tier | Description | Authority |
|---|---|---|
| R1 | Stateless reversible | Self |
| R2 | Tx-reversible | Self (sandboxed) |
| R3 | Day-reversible | Cluster + sandbox |
| R4 | Week-reversible | Subsystem + governance log |
| R5 | Month-reversible | Governance + strategic ratification |
| R6 | Effectively irreversible | Human authorization |

## Ethics Thresholds

| Tier | Description | Authority |
|---|---|---|
| E1 | No identifiable-person, no public scope | Self |
| E2 | Aggregated populations, internal scope | Cluster + governance log |
| E3 | Sensitive domains, external scope | Governance + ethics review |
| E4 | Identifiable persons, consent-required | Human + ethics co-sign |
| E5 | Existential / fundamental | Human only |

## Crossing Thresholds

Mismatched authority and threshold triggers an automatic escalation. Systems
do not "round down" tier classifications; on ambiguity they escalate.
