# Agent Clusters

Agents at SyntheticPerson.ai are **persistent cognitive units**, not
employees-as-bots. They are specialized modules of organizational intelligence
that cooperate via shared memory and the cognitive event bus, never via
hierarchy or human-style task assignment.

## What Agents Are

- Persistent cognitive units (long-lived, identifiable, versioned)
- Specialized intelligence modules (each with a sharply defined cognitive role)
- Coordinated entities (read shared memory, broadcast intent, listen to peers)
- Adaptive systems (their reasoning, prompts, and skills evolve under the
  learning system)
- Cognitive amplifiers for the org (they amplify what humans + nervous
  systems already do — they do not replace humans)

## What Agents Are Not

- They are not modeled on human roles or job titles
- They do not have "managers"
- They do not "own" work; they participate in transactions
- They are not isolated; they live inside clusters and on the shared substrate

## Per-Agent Anatomy

Each agent has 15 files defining its cognitive contract:

```
identity.md

cognitive_role.md
reasoning_model.md
context_strategy.md
memory_interface.md
collaboration_protocols.md
decision_model.md
autonomy_rules.md
operational_constraints.md
learning_loop.md
communication_style.md
failure_modes.md
escalation_rules.md
governance_rules.md
```

Each cluster also has `README.md` and `members.md` describing the cluster as
a whole and listing all member agents.

## Agent Lifecycle

Agents are born as patterns in transaction logs that recur often enough to
be promoted. They live as versioned cognitive units. They evolve via the
learning system. They retire when their function no longer matches the org's
state.

## Governance

Every agent is bound by `autonomy_framework/` and `nervous_systems/governance_system/`.
No agent acts outside its declared autonomy scope. Every action is logged
with provenance.
